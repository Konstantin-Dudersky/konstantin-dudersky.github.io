Все каналы подключены через I²C-мультиплексор **TCA9548A** [^1], что позволяет подключать к разным каналам периферию с одинаковым адресом.

Все устройства гальванически изолированы от шины IBus с помощью I²C-изолятора **CA-IS3021** [^2].

Подключённые устройства можно запитывать как 5В, так и 3.3В. Напряжение питания одинаковое для всех каналов.

## Схема внешних подключений

<ExtConn />

Таблица с перечнем коннекторов:

| Коннектор | Описание  |
|-----------|-----------|
| XG00      | Канал 0   |
| XG01      | Канал 1   |
| XG02      | Канал 2   |
| XG03      | Канал 3   |
| XG04      | Канал 4   |
| XG05      | Канал 5   |
| XG06      | Канал 6   |
| XG07      | Канал 7   |
| XG08      | Выходное питание |

К модулю **не надо** подключать внешнее питание. Клемма L+ - это выходное питание с внутреннего блока питания.

## Адрес I²C

Адрес чипа **TCA9548A** задаётся тремя пинами A0, A1, A2. Пины могут быть подтянуты к питанию или к земле. В зависимости от этого можно задать один из восьми возможных адресов.

| Адрес | A2 | A1 | A0 |
|-------|----|----|----|
| 0x70  | -  | -  | -  |
| 0x71  | -  | -  | +  |
| 0x72  | -  | +  | -  |
| 0x73  | -  | +  | +  |
| 0x74  | +  | -  | -  |
| 0x75  | +  | -  | +  |
| 0x76  | +  | +  | -  |
| 0x77  | +  | +  | +  |

## Описание

<Schematic />

Питание подключаемых устройств осуществляется через изолированный блок питания **B2405** или **B2403** мощностью от 1 до 3 Вт.

Шина I²C гальванически изолирована от шины IBus с помощью I²C-изолятора **CA-IS3021** [^2]. Рабочее напряжение логики на разных сторонах изолятора может быть разным (как 3,3 В, так и 5 В). Поэтому к данному модулю можно подключать периферийные устройства с работой от 5 В, даже несмотря на то, что модули контроллеров работают с логикой 3,3 В.

Отдельные каналы подключены через I²C-мультиплексор **TCA9548A** [^1]. Поэтому к разным каналам можно подключать устройства с одинаковым адресом I²C.

Нумерация каналов чипа **TCA9548A** не совпадает с нумерацией каналов на модуле - в приоритетах разводки платы было использование 2 слоев с минимум переходных отверстий.

## Примеры работы в Linux

При использовании контроллера, работающего на ОС Linux, можно проверить работу модуля из консоли.

В примере принято:
- адрес чипа **TCA9548A** 0x70
- устройство Linux `/dev/i2c-0`

```sh
# Проверить доступность
i2cdetect -y 0

# Открыть каналы TCA9548A с 0 по 7
i2ctransfer -y 0 w1@0x70 0x01
i2ctransfer -y 0 w1@0x70 0x02
i2ctransfer -y 0 w1@0x70 0x04
i2ctransfer -y 0 w1@0x70 0x08
i2ctransfer -y 0 w1@0x70 0x10
i2ctransfer -y 0 w1@0x70 0x20
i2ctransfer -y 0 w1@0x70 0x40
i2ctransfer -y 0 w1@0x70 0x80

# Открыть все каналы
i2ctransfer -y 0 w1@0x70 0xFF

# Закрыть все каналы
i2ctransfer -y 0 w1@0x70 0x00
```
