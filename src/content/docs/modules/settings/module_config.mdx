---
title: Настройка модуля
description: Описание подключения к модулю с микроконтроллером ESP32-C3
---

import { Tabs, TabItem, Steps } from '@astrojs/starlight/components';

Подключение и настройка модуля выполняется с помощью утилиты **espflash**.

<Steps>

1. Перейти в репозиторий [GitHub](https://github.com/esp-rs/espflash/releases).

2. Скачать исполняемый файл актуальной версии для своей операционной системы:

    <Tabs syncKey="os">
        <TabItem label="Linux">
            `espflash-x86_64-unknown-linux-gnu.zip`
        </TabItem>
        <TabItem label="Windows">
            `espflash-x86_64-pc-windows-msvc.zip`
        </TabItem>
    </Tabs>

3. Распаковываем архив. Утилита работает без установки. Для удобства можно прописать путь к переменной среды **PATH**.

    <Tabs syncKey="os">
        <TabItem label="Linux">
            Для оболочки **bash** необходимо добавить в конец файла `~/.bashrc` следующую строку:

            ```sh
            export PATH="$PATH:/путь/к/директории"
            ```
        </TabItem>
        <TabItem label="Windows">
            - Нажать **Win+R**
            - Ввести `sysdm.cpl`
            - Выбрать вкладку **Дополнительно**, нажать кнопку **Переменные среды**
            - Добавить в переменную среды **PATH** путь к каталогу с распакованной программой **espflash**
        </TabItem>
    </Tabs>

    Если не редактировать переменную среды **PATH**, после запуска терминала необходимо перейти в каталог с распакованной программой **espflash**.

4. Подключаем модуль к компьютеру через USB-кабель. Подаём питание на модуль.

5. Запускаем терминал. Запускаем команду `espflash monitor`.

    <Tabs syncKey="os">
        <TabItem label="Linux">
            ```sh
            espflash monitor
            ```
        </TabItem>
        <TabItem label="Windows">
            ```sh
            espflash.exe monitor
            ```
        </TabItem>
    </Tabs>

6. В терминале выведется примерно следующее:

    <Tabs syncKey="os">
        <TabItem label="Linux">
            ```sh
            [2026-07-11T07:21:01Z INFO ] Serial port: '/dev/ttyACM0'
            [2026-07-11T07:21:01Z INFO ] Connecting...
            [2026-07-11T07:21:01Z INFO ] Using flash stub
            Commands:
                CTRL+R    Reset chip
                CTRL+C    Exit
            ```
        </TabItem>
        <TabItem label="Windows">
            ```sh
            [2026-07-11T07:10:59Z INFO ] Serial port: 'COM6'
            [2026-07-11T07:10:59Z INFO ] Connecting...
            [2026-07-11T07:11:00Z INFO ] Using flash stub
            Commands:
                CTRL+R    Reset chip
                CTRL+C    Exit
            ```
        </TabItem>
    </Tabs>

    Это значит, что соединение с модулем установлено.

7. Нажимаем `CTRL+R` в терминале. При этом микроконтроллер перезагрузится.

8. Когда в терминале появится сообщение:

    ```
    Нажмите BOOT для изменения настроек
    ```

    то необходимо в течение 5 секунд нажать кнопку BOOT на модуле. Если не нажать, то микроконтроллер продолжит обычную загрузку.

9. Далее необходимо выполнить необходиму настройку. Перечень настроек зависит от конкретного модуля. Для вывода справки введите в терминале:

    ```sh
    config --help
    ```

    Для вывода справки по конкретной команде (например, `config can`), введите:

    ```sh
    config can --help
    ```

10. Для сохранения настроек введите команду:

    ```sh
    config save
    ```

    Микроконтроллер сохранит настройки в энергонезависимой памяти и продолжит обычную загрузку.

11. Для выхода без сохранения настроек введите:

    ```sh
    config exit
    ```

    Микроконтроллер продолжит обычную загрузку.

    Для выхода из настроек можно просто перезагрузить модуль.

</Steps>
